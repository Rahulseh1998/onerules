# Contributing to onerules

Thanks for your interest in contributing! Here's how to add support for new frameworks, tools, and languages.

## Adding a New Framework

1. Create a new file in `src/templates/fragments/`:

```typescript
// src/templates/fragments/myframework.ts
import type { RuleSet } from "../../types.js";

export function getMyframeworkRules(): RuleSet {
  return {
    projectContext: "This is a MyFramework application.",
    codingPatterns: [
      // Framework-specific coding patterns
    ],
    architecture: [
      // Architecture and project structure rules
    ],
    doNot: [
      // Common mistakes to avoid
    ],
  };
}
```

2. Register it in `src/templates/fragments/index.ts`:

```typescript
import { getMyframeworkRules } from "./myframework.js";

// Add to the frameworkRules map:
const frameworkRules = {
  // ...existing
  myframework: getMyframeworkRules,
};
```

3. Add the framework type to `src/types.ts` if it's not already there.

4. Add detection logic in the appropriate detector file (`src/detect/javascript.ts`, etc.).

5. Run `npm test` and `npx tsc --noEmit` to verify.

## Adding a New AI Tool

1. Create a generator in `src/generate/`:

```typescript
// src/generate/mytool.ts
import type { StackProfile, ToolOutput } from "../types.js";
import { buildRuleSet, renderMarkdownRules, formatStackSummary } from "./common.js";

export function generateMytool(profile: StackProfile): ToolOutput {
  const rules = buildRuleSet(profile);
  const header = `# MyTool Rules\n\n> ${formatStackSummary(profile)} — generated by onerules`;
  const content = renderMarkdownRules(rules, header);

  return {
    toolId: "mytool",
    toolName: "MyTool",
    filePath: ".mytoolrules",
    content,
  };
}
```

2. Add the tool ID to `ToolId` type in `src/types.ts`.

3. Register the generator in `src/generate/index.ts`.

4. Update tests in `src/generate/generate.test.ts`.

## Guidelines

- **Rules should be principle-based**, not prescriptive. "Use Server Components by default" is better than "Always add 'use server' to the top of every file."
- **Keep rules concise.** Each rule should be one sentence. No paragraphs.
- **Focus on what's unique to the framework.** Don't repeat general coding advice.
- **Include "Do Not" rules** for the most common mistakes.
- **Test with a real project** if possible.

## Development

```bash
npm install          # Install dependencies
npm run build        # Build
npm test             # Run tests
npx tsc --noEmit     # Type check
node dist/cli.js     # Run locally
```
